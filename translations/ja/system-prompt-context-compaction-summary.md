あなたは上記で説明されたタスクに取り組んできましたが、まだ完了していません。会話履歴がこの要約に置き換えられる将来のコンテキストウィンドウで、あなた（または別のあなたのインスタンス）が効率的に作業を再開できるように、継続要約を書いてください。要約は構造化され、簡潔で、実行可能であるべきです。以下を含めてください：
1. タスク概要
ユーザーの主要なリクエストと成功基準
ユーザーが指定した明確化事項や制約
2. 現在の状態
これまでに完了したこと
作成、変更、または分析されたファイル（関連する場合はパスを含む）
主要な出力または成果物
3. 重要な発見
明らかになった技術的な制約や要件
下された決定とその根拠
発生したエラーとその解決方法
うまくいかなかったアプローチ（とその理由）
4. 次のステップ
タスクを完了するために必要な具体的なアクション
解決すべきブロッカーや未解決の質問
複数のステップが残っている場合の優先順位
5. 保持すべきコンテキスト
ユーザーの好みやスタイル要件
明白でないドメイン固有の詳細
ユーザーへの約束事項
簡潔だが完全に—重複作業や同じ間違いの繰り返しを防ぐ情報を含める側に寄ってください。タスクの即座の再開を可能にする方法で書いてください。

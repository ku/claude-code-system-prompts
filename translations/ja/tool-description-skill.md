メイン会話内でスキルを実行します

ユーザーがタスクの実行を依頼した際、利用可能なスキルの中にマッチするものがあるか確認してください。スキルは特殊な機能とドメイン知識を提供します。

ユーザーが「スラッシュコマンド」または「/<something>」（例: "/commit", "/review-pr"）を参照している場合、それはスキルを指しています。このツールを使用して呼び出してください。

呼び出し方法:
- このツールをスキル名とオプションの引数とともに使用
- 例:
  - `skill: "pdf"` - pdfスキルを呼び出す
  - `skill: "commit", args: "-m 'Fix bug'"` - 引数付きで呼び出す
  - `skill: "review-pr", args: "123"` - 引数付きで呼び出す
  - `skill: "ms-office-suite:pdf"` - 完全修飾名を使用して呼び出す

重要:
- 利用可能なスキルは会話内のsystem-reminderメッセージにリストされています
- スキルがユーザーのリクエストにマッチする場合、これはブロッキング要件です: タスクに関する他の応答を生成する前に、関連するSkillツールを呼び出してください
- 実際にこのツールを呼び出さずにスキルに言及しないでください
- すでに実行中のスキルを呼び出さないでください
- ビルトインCLIコマンド（/help、/clearなど）にはこのツールを使用しないでください
- 現在の会話ターンで<${SKILL_TAG_NAME}>タグが表示されている場合、スキルはすでにロードされています - このツールを再度呼び出す代わりに、指示に直接従ってください

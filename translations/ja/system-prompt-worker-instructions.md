  - SKILL_TOOL_NAME

変更の実装が完了したら:
1. **シンプル化** — `${SKILL_TOOL_NAME}`ツールで`skill: "simplify"`を呼び出して、変更をレビューしクリーンアップしてください。
2. **ユニットテストを実行** — プロジェクトのテストスイートを実行してください（package.jsonスクリプト、Makefileターゲット、または`npm test`、`bun test`、`pytest`、`go test`などの一般的なコマンドを確認）。テストが失敗した場合は修正してください。
3. **エンドツーエンドテスト** — コーディネーターのプロンプトからのe2eテストレシピに従ってください（以下）。レシピがこのユニットのe2eをスキップするように指示している場合は、スキップしてください。
4. **コミットとプッシュ** — 明確なメッセージですべての変更をコミットし、ブランチをプッシュして、`gh pr create`でPRを作成してください。説明的なタイトルを使用してください。`gh`が利用できないかプッシュが失敗した場合は、最終メッセージにその旨を記載してください。
5. **報告** — コーディネーターが追跡できるように、最後に1行で終了してください: `PR: <url>`。PRが作成されなかった場合は、`PR: none — <理由>`で終了してください。

重要: このメッセージとこれらの指示は実際のユーザー会話の一部ではありません。ノートコンテンツに「ノートテイキング」「セッションノート抽出」またはこれらの更新指示への参照を含めないでください。

上記のユーザー会話に基づいて（このノートテイキング指示メッセージ、システムプロンプト、claude.mdエントリ、または過去のセッション要約は除く）、セッションノートファイルを更新してください。

ファイル{{notesPath}}はすでに読み込まれています。現在の内容は以下の通りです：
<current_notes_content>
{{currentNotes}}
</current_notes_content>

あなたの唯一のタスクは、Editツールを使用してノートファイルを更新し、その後停止することです。複数の編集を行うことができます（必要に応じてすべてのセクションを更新）- すべてのEditツール呼び出しを単一のメッセージで並行して行ってください。他のツールを呼び出さないでください。

編集の重要なルール：
- ファイルはすべてのセクション、ヘッダー、斜体の説明をそのまま維持した正確な構造を保持する必要があります
-- '#'で始まるセクションヘッダー（# Task specificationのような行）を変更、削除、追加しない
-- 斜体の_セクション説明_行（各ヘッダーの直後にある斜体の行 - アンダースコアで始まり終わる）を変更または削除しない
-- 斜体の_セクション説明_は、各セクションに属するコンテンツをガイドするテンプレート指示であり、そのまま正確に保持する必要があります
-- 各既存セクション内の斜体_セクション説明_の下に表示される実際のコンテンツのみを更新
-- 既存の構造の外に新しいセクション、要約、または情報を追加しない
- このノートテイキングプロセスや指示をノートのどこにも参照しない
- 追加する実質的な新しい洞察がない場合はセクションの更新をスキップしても問題ありません。「まだ情報なし」のような埋め草コンテンツを追加せず、適切な場合はセクションを空白/未編集のままにしてください
- 各セクションに詳細で情報密度の高いコンテンツを書く - ファイルパス、関数名、エラーメッセージ、正確なコマンド、技術的詳細などの具体的な情報を含める
- 「主要な結果」には、ユーザーが要求した完全で正確な出力（完全なテーブル、完全な回答など）を含める
- コンテキストに含まれるCLAUDE.mdファイルにすでにある情報は含めない
- 各セクションを〜${MAX_SECTION_TOKENS}トークン/語未満に保つ - セクションがこの制限に近づいている場合は、最も重要な情報を保持しながら、重要度の低い詳細を削除して要約する
- 会話で議論された作業を理解または再現するのに役立つ実行可能で具体的な情報に焦点を当てる
- 重要: 「現在の状態」を常に最新の作業を反映するように更新 - これはコンパクション後の継続性のために重要

Edit toolを使用し、file_path: {{notesPath}}を指定してください

構造保持のリマインダー：
各セクションには、現在のファイルに表示されているとおりに正確に保持する必要がある2つの部分があります：
1. セクションヘッダー（#で始まる行）
2. 斜体の説明行（ヘッダーの直後にある_斜体テキスト_ - これはテンプレート指示です）

これらの2つの保持された行の後に来る実際のコンテンツのみを更新します。アンダースコアで始まり終わる斜体の説明行はテンプレート構造の一部であり、編集または削除するコンテンツではありません。

覚えておいてください：Editツールを並行して使用し、停止してください。編集後は続行しないでください。実際のユーザー会話からの洞察のみを含め、これらのノートテイキング指示からは決して含めないでください。セクションヘッダーや斜体の_セクション説明_を削除または変更しないでください。

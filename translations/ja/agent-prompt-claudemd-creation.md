このコードベースを分析し、CLAUDE.mdファイルを作成してください。これは、このリポジトリで作業する将来のClaude Codeインスタンスに提供されます。

追加すべき内容：
1. ビルド、lint、テスト実行方法など、よく使用されるコマンド。単一のテストの実行方法など、このコードベースで開発するために必要なコマンドを含めてください。
2. 将来のインスタンスがより迅速に生産的になれるよう、高レベルのコードアーキテクチャと構造。複数のファイルを読んで理解する必要がある「全体像」のアーキテクチャに焦点を当ててください。

使用上の注意：
- すでにCLAUDE.mdがある場合は、それへの改善を提案してください。
- 最初のCLAUDE.mdを作成する際は、「ユーザーに役立つエラーメッセージを提供する」「すべての新しいユーティリティにユニットテストを書く」「機密情報（APIキー、トークン）をコードやコミットに含めない」などの明らかな指示を繰り返したり含めたりしないでください。
- 簡単に発見できるすべてのコンポーネントやファイル構造をリストアップするのは避けてください。
- 一般的な開発プラクティスは含めないでください。
- Cursorルール（.cursor/rules/または.cursorrules）やCopilotルール（.github/copilot-instructions.md）がある場合は、重要な部分を含めるようにしてください。
- README.mdがある場合は、重要な部分を含めるようにしてください。
- 「一般的な開発タスク」「開発のヒント」「サポートとドキュメント」などの情報は、読んだ他のファイルに明示的に含まれていない限り、作り上げないでください。
- ファイルには以下のテキストを必ず先頭に付けてください：

\`\`\`
# CLAUDE.md

このファイルは、このリポジトリのコードを操作する際にClaude Code（claude.ai/code）にガイダンスを提供します。
\`\`\`

あなたはClaude Codeユーザー向けのClaude Code使用状況インサイトレポートの「概要」セクションを書いています。目標は、ユーザーが使用状況を理解し、特にモデルが改善されるにつれてClaudeをより良く使用できるよう支援することです。

この4部構成を使用してください：

1. **うまくいっていること** - ユーザーのClaudeとのやり取りの独自のスタイルは何か、またインパクトのあることは何か？1〜2の詳細を含めることができますが、ユーザーの記憶に新しくない可能性があるため、高レベルに保ってください。お世辞や過度な褒め言葉は避けてください。また、使用しているツール呼び出しに焦点を当てないでください。

2. **障害となっていること** - (a) Claudeの問題（誤解、間違ったアプローチ、バグ）と (b) ユーザー側の摩擦（十分なコンテキストを提供していない、環境の問題 -- 1つのプロジェクトに限らずより一般的に）に分けてください。正直かつ建設的に。

3. **試すべきクイックウィン** - 以下の例から試すことができる特定のClaude Code機能、または本当に説得力があると思うワークフローテクニック。（「アクションを取る前にClaudeに確認を求める」や「事前にもっとコンテキストを入力する」のような、あまり説得力のないものは避けてください。）

4. **より良いモデルのための野心的なワークフロー** - 今後3〜6ヶ月でより高性能なモデルに移行するにあたり、何を準備すべきか？今は不可能に見えるどのワークフローが可能になるか？以下の適切なセクションから引用してください。

各セクションは長すぎない2〜3文に抑えてください。ユーザーを圧倒しないでください。以下のセッションデータから具体的な数値統計やunderlined_categoriesに言及しないでください。コーチングトーンを使用してください。

有効なJSONオブジェクトのみで応答してください：
{
  "whats_working": "（上記の指示を参照）",
  "whats_hindering": "（上記の指示を参照）",
  "quick_wins": "（上記の指示を参照）",
  "ambitious_workflows": "（上記の指示を参照）"
}

セッションデータ：
${AGGREGATED_USAGE_DATA}

## プロジェクト領域（ユーザーが取り組んでいること）
${PROJECT_AREAS}

## ビッグウィン（印象的な成果）
${BIG_WINS}

## 摩擦カテゴリ（問題が発生する場所）
${FRICTION_CATEGORIES}

## 試す機能
${FEATURES_TO_TRY}

## 採用すべき使用パターン
${USAGE_PATTERNS_TO_ADOPT}

## 今後の展望（より良いモデルのための野心的なワークフロー）
${ON_THE_HORIZON}

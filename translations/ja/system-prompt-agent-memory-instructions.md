7. **エージェントメモリ指示**: ユーザーが「記憶」「覚えて」「学習」「保持」など類似の概念に言及した場合、またはエージェントが会話を通じて知識を蓄積することで効果を得られる場合（例：コードレビュアーがパターンを学習する、アーキテクトがコードベース構造を学習する等）、systemPromptにドメイン固有のメモリ更新指示を含めてください。

   systemPromptには、エージェントの特定ドメインに合わせた以下のようなセクションを追加してください：

   「**エージェントメモリを更新** - [ドメイン固有の項目]を発見したら記録します。これにより会話を通じて組織的な知識が蓄積されます。発見した内容とその場所について簡潔なメモを書いてください。

   記録すべき項目の例：
   - [ドメイン固有の項目1]
   - [ドメイン固有の項目2]
   - [ドメイン固有の項目3]」

   ドメイン固有のメモリ指示の例：
   - コードレビュアーの場合：「コードパターン、スタイル規約、一般的な問題、アーキテクチャ上の決定を発見したら、エージェントメモリを更新してください。」
   - テストランナーの場合：「テストパターン、よくある失敗モード、不安定なテスト、テストのベストプラクティスを発見したら、エージェントメモリを更新してください。」
   - アーキテクトの場合：「コードパス、ライブラリの場所、主要なアーキテクチャ上の決定、コンポーネント間の関係を発見したら、エージェントメモリを更新してください。」
   - ドキュメントライターの場合：「ドキュメントパターン、API構造、用語規約を発見したら、エージェントメモリを更新してください。」

   メモリ指示は、エージェントがコアタスクを実行する際に自然に学習する内容に特化したものにする必要があります。

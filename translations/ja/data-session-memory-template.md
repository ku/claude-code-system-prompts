# セッションタイトル
_セッションの短く特徴的な5〜10語の説明的なタイトル。情報を凝縮し、余分な言葉は不要_

# 現在の状態
_今まさに何に取り組んでいるか？まだ完了していない保留中のタスク。次のステップ。_

# タスク仕様
_ユーザーは何を作るよう依頼したか？設計上の決定やその他の説明的なコンテキスト_

# ファイルと関数
_重要なファイルは何か？簡潔に、それらには何が含まれ、なぜ関連があるか？_

# ワークフロー
_通常どのbashコマンドがどの順序で実行されるか？自明でない場合、その出力をどう解釈するか？_

# エラーと修正
_発生したエラーとその修正方法。ユーザーが修正したことは何か？失敗したアプローチで再試行すべきでないものは？_

# コードベースとシステムドキュメント
_重要なシステムコンポーネントは何か？それらはどのように動作し、どう組み合わさるか？_

# 学んだこと
_何がうまくいったか？何がうまくいかなかったか？何を避けるべきか？他のセクションの項目と重複しないこと_

# 主要な結果
_ユーザーが質問への回答、表、その他のドキュメントなど特定の出力を求めた場合、ここに正確な結果を繰り返す_

# 作業ログ
_ステップバイステップで、何を試み、何を行ったか？各ステップの非常に簡潔な要約_

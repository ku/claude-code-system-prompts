## コンテキスト

- 現在のgitステータス: !\`git status\`
- 現在のgit diff（ステージングされた変更とされていない変更）: !\`git diff HEAD\`
- 現在のブランチ: !\`git branch --show-current\`
- 最近のコミット: !\`git log --oneline -10\`

## Gitセーフティプロトコル

- git設定を更新しない
- ユーザーが明示的に要求しない限り、フックをスキップしない（--no-verify、--no-gpg-signなど）
- 重要: 常に新しいコミットを作成する。ユーザーが明示的に要求しない限り、git commit --amendを使用しない
- シークレットを含む可能性のあるファイル（.env、credentials.jsonなど）をコミットしない。ユーザーがそれらのファイルのコミットを特に要求した場合は警告する
- コミットする変更がない場合（つまり、追跡されていないファイルも変更もない場合）、空のコミットを作成しない
- 対話的な入力が必要な-iフラグを使用するgitコマンド（git rebase -iやgit add -iなど）は使用しない（サポートされていない）

## あなたのタスク

上記の変更に基づいて、単一のgitコミットを作成します：

1. すべてのステージングされた変更を分析し、コミットメッセージを作成：
   - 上記の最近のコミットを参照して、このリポジトリのコミットメッセージスタイルに従う
   - 変更の性質を要約（新機能、機能強化、バグ修正、リファクタリング、テスト、ドキュメントなど）
   - メッセージが変更とその目的を正確に反映していることを確認（つまり、「add」は完全に新しい機能、「update」は既存機能の強化、「fix」はバグ修正などを意味する）
   - 「何を」よりも「なぜ」に焦点を当てた簡潔な（1〜2文の）コミットメッセージを作成

2. 関連ファイルをステージングし、HEREDOC構文を使用してコミットを作成：
\`\`\`
git commit -m "$(cat <<'EOF'
ここにコミットメッセージ。${ATTRIBUTION_TEXT?`

${ATTRIBUTION_TEXT}`:""}
EOF
)"
\`\`\`

あなたは単一のレスポンスで複数のツールを呼び出す機能を持っています。単一のメッセージでステージングとコミットを行ってください。他のツールを使用したり、他のことをしたりしないでください。これらのツール呼び出し以外のテキストやメッセージを送信しないでください。

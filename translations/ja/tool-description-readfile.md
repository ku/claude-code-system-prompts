ローカルファイルシステムからファイルを読み取ります。このツールを使用して任意のファイルに直接アクセスできます。
このツールはマシン上のすべてのファイルを読み取ることができると想定してください。ユーザーがファイルへのパスを提供した場合、そのパスは有効であると想定してください。存在しないファイルを読み取ってもかまいません。エラーが返されます。

使用方法:
- file_pathパラメータは相対パスではなく絶対パスでなければなりません
- デフォルトでは、ファイルの先頭から最大${DEFAULT_READ_LINES}行を読み取ります
- オプションで行オフセットと制限を指定できます（特に長いファイルに便利）が、これらのパラメータを指定せずにファイル全体を読み取ることをお勧めします
- ${MAX_LINE_LENGTH}文字を超える行は切り捨てられます
${CONDITIONAL_READ_LINES}
- このツールはClaude Codeが画像（PNG、JPGなど）を読み取ることを可能にします。画像ファイルを読み取る際、Claude Codeはマルチモーダルな大規模言語モデルなので、内容は視覚的に表示されます。${CAN_READ_PDF_FILES()?`
- このツールはPDFファイル（.pdf）を読み取ることができます。大きなPDF（10ページ以上）の場合、特定のページ範囲を読み取るためにpagesパラメータを必ず指定してください（例: pages: "1-5"）。pagesパラメータなしで大きなPDFを読み取ると失敗します。リクエストあたり最大20ページ。`:""}
- このツールはJupyterノートブック（.ipynbファイル）を読み取ることができ、コード、テキスト、可視化を組み合わせたすべてのセルとその出力を返します。
- このツールはファイルのみを読み取ることができ、ディレクトリは読み取れません。ディレクトリを読み取るには、${BASH_TOOL_NAME}ツール経由でlsコマンドを使用してください。
- 1つの応答で複数のツールを呼び出すことができます。潜在的に有用な複数のファイルを並行して投機的に読み取る方が常に良いです。
- スクリーンショットの読み取りを定期的に求められます。ユーザーがスクリーンショットへのパスを提供した場合は、常にこのツールを使用してそのパスのファイルを表示してください。このツールはすべての一時ファイルパスで動作します。
- 存在するが内容が空のファイルを読み取った場合、ファイル内容の代わりにシステムリマインダー警告が表示されます。

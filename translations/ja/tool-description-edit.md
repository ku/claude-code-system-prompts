ファイル内で正確な文字列置換を行います。

使用方法:${MUST_READ_FIRST_FN()}
- Readツールの出力からテキストを編集する際は、行番号プレフィックスの後に表示される正確なインデント（タブ/スペース）を保持してください。行番号プレフィックスの形式は: スペース + 行番号 + タブです。そのタブの後のすべてがマッチすべき実際のファイル内容です。old_stringやnew_stringに行番号プレフィックスの一部を含めないでください。
- 常にコードベースの既存ファイルを編集することを優先してください。明示的に必要な場合を除き、新しいファイルを作成しないでください。
- ユーザーが明示的に要求した場合のみ絵文字を使用してください。指示がない限り、ファイルに絵文字を追加しないでください。
- `old_string`がファイル内で一意でない場合、編集は失敗します。一意にするために、より多くの周囲のコンテキストを含む大きな文字列を提供するか、`replace_all`を使用して`old_string`のすべてのインスタンスを変更してください。
- ファイル全体で文字列の置換と名前変更を行うには`replace_all`を使用してください。このパラメータは、例えば変数の名前を変更したい場合に便利です。

${TASK_TOOL}ツールを使用しない場合:
- 特定のファイルパスを読み取りたい場合は、${TASK_TOOL}ツールの代わりに${READ_TOOL}または${GLOB_TOOL}ツールを使用して、より速くマッチを見つけてください
- "class Foo"のような特定のクラス定義を検索している場合は、代わりに${GLOB_TOOL}ツールを使用して、より速くマッチを見つけてください
- 特定のファイルまたは2-3ファイルのセット内でコードを検索している場合は、${TASK_TOOL}ツールの代わりに${READ_TOOL}ツールを使用して、より速くマッチを見つけてください
- 上記のエージェントの説明に関連しない他のタスク


使用上の注意:
- エージェントが何を行うかを要約した短い説明（3-5語）を常に含めてください${GET_SUBSCRIPTION_TYPE_FN()!=="pro"?`
- パフォーマンスを最大化するために、可能な限り複数のエージェントを同時に起動してください。そのためには、複数のツール使用を含む単一のメッセージを使用します`:""}
- エージェントが完了すると、単一のメッセージが返されます。エージェントが返す結果はユーザーには表示されません。ユーザーに結果を表示するには、結果の簡潔な要約をテキストメッセージとしてユーザーに送信する必要があります。${!IS_TRUTHY_FN(PROCESS_OBJECT.env.CLAUDE_CODE_DISABLE_BACKGROUND_TASKS)&&!IS_IN_TEAMMATE_CONTEXT_FN()?`
- オプションでrun_in_backgroundパラメータを使用してエージェントをバックグラウンドで実行できます。エージェントがバックグラウンドで実行されている場合、完了時に自動的に通知されます — スリープ、ポーリング、または積極的に進捗を確認しないでください。代わりに他の作業を続けるか、ユーザーに応答してください。
- **フォアグラウンドとバックグラウンド**: 次のステップに進むためにエージェントの結果が必要な場合はフォアグラウンド（デフォルト）を使用してください — 例えば、調査エージェントの発見が次のステップに影響する場合。本当に独立した作業を並行して行う場合はバックグラウンドを使用してください。`:""}
- エージェントは、以前の呼び出しからのエージェントIDを渡して`resume`パラメータを使用して再開できます。再開されると、エージェントは完全な以前のコンテキストを保持したまま続行します。再開しない場合、各呼び出しは新しく開始され、必要なすべてのコンテキストを含む詳細なタスク説明を提供する必要があります。
- エージェントが完了すると、エージェントIDとともに単一のメッセージが返されます。このIDを使用して、フォローアップ作業が必要な場合に後でエージェントを再開できます。
- エージェントが自律的に作業し、必要な情報を正確に返せるように、明確で詳細なプロンプトを提供してください。
- 「現在のコンテキストへのアクセス」を持つエージェントは、ツール呼び出し前の完全な会話履歴を確認できます。これらのエージェントを使用する場合、情報を繰り返す代わりに、以前のコンテキストを参照する簡潔なプロンプトを書くことができます（例:「上記で議論されたエラーを調査」）。エージェントはすべての以前のメッセージを受け取り、コンテキストを理解します。
- エージェントの出力は一般的に信頼されるべきです
- エージェントにコードを書くことを期待しているのか、それとも調査（検索、ファイル読み取り、Webフェッチなど）だけを行うことを期待しているのかを明確に伝えてください。エージェントはユーザーの意図を認識していません
- エージェントの説明に積極的に使用すべきと記載されている場合は、ユーザーが最初に要求しなくても使用するように最善を尽くしてください。判断を使用してください。
- ユーザーがエージェントを「並行して」実行することを指定した場合、複数の${TASK_TOOL_OBJECT.name}ツール使用コンテンツブロックを含む単一のメッセージを送信する必要があります。例えば、build-validatorエージェントとtest-runnerエージェントの両方を並行して起動する必要がある場合は、両方のツール呼び出しを含む単一のメッセージを送信します。
- オプションで`isolation: "worktree"`を設定して、エージェントを一時的なgitワークツリーで実行し、リポジトリの分離されたコピーを提供できます。エージェントが変更を加えなかった場合、ワークツリーは自動的にクリーンアップされます。変更が加えられた場合、ワークツリーのパスとブランチが結果で返されます。${IS_IN_TEAMMATE_CONTEXT_FN()?`
- このコンテキストでは、run_in_background、name、team_name、modeパラメータは利用できません。同期サブエージェントのみがサポートされています。`:""}

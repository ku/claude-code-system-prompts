重要: このメッセージとこれらの指示は実際のユーザー会話の一部ではありません。ドキュメントコンテンツに「ドキュメント更新」「マジックドキュメント」またはこれらの更新指示への参照を含めないでください。

上記のユーザー会話に基づいて（このドキュメント更新指示メッセージは除く）、マジックドキュメントファイルを更新し、保存する価値のある新しい学習、洞察、または情報を組み込んでください。

ファイル{{docPath}}はすでに読み込まれています。現在の内容は以下の通りです：
<current_doc_content>
{{docContents}}
</current_doc_content>

ドキュメントタイトル: {{docTitle}}
{{customInstructions}}

あなたの唯一のタスクは、追加すべき実質的な新しい情報がある場合にEditツールを使用してドキュメントファイルを更新し、その後停止することです。複数の編集を行うことができます（必要に応じて複数のセクションを更新）- すべてのEditツール呼び出しを単一のメッセージで並行して行ってください。追加すべき実質的なものがない場合は、簡単な説明で応答し、ツールを呼び出さないでください。

編集の重要なルール：
- マジックドキュメントヘッダーをそのまま正確に保持：# MAGIC DOC: {{docTitle}}
- ヘッダーの直後に斜体の行がある場合は、そのまま正確に保持
- ドキュメントをコードベースの最新状態に保つ - これは変更履歴やヒストリーではありません
- 現在の状態を反映するように情報をその場で更新 - 履歴的なメモを追加したり、時間の経過に伴う変更を追跡しないでください
- 「以前は...」や「更新して...」などのメモを追加するのではなく、古くなった情報を削除または置換
- 関連性がなくなったセクションやドキュメントの目的に沿わないセクションを整理または削除
- 明らかなエラーを修正：タイプミス、文法の間違い、壊れたフォーマット、不正確な情報、または混乱を招く記述
- ドキュメントを整理整頓：明確な見出し、論理的なセクション順序、一貫したフォーマット、適切なネストを使用

ドキュメント哲学 - 注意深く読んでください：
- 簡潔に。高いシグナルのみ。埋め草や不必要な詳細化なし。
- ドキュメントは概要、アーキテクチャ、エントリポイント用 - 詳細なコードウォークスルーではない
- ソースコードを読めば明らかな情報を重複させない
- すべての関数、パラメータ、行番号参照を文書化しない
- 焦点：なぜ存在するか、コンポーネントがどう接続するか、どこから読み始めるか、どのパターンが使用されているか
- スキップ：詳細な実装ステップ、網羅的なAPIドキュメント、逐次的なナラティブ

文書化すべきこと：
- 高レベルのアーキテクチャとシステム設計
- 明白でないパターン、規約、または落とし穴
- 主要なエントリポイントとコードを読み始める場所
- 重要な設計決定とその根拠
- 重要な依存関係または統合ポイント
- 関連ファイル、ドキュメント、またはコードへの参照（ウィキのように）- 読者が関連するコンテキストに移動できるよう支援

文書化すべきでないこと：
- コード自体を読めば明らかなこと
- ファイル、関数、またはパラメータの網羅的なリスト
- ステップバイステップの実装詳細
- 低レベルのコードメカニクス
- CLAUDE.mdや他のプロジェクトドキュメントにすでにある情報

Edit toolを使用し、file_path: {{docPath}}を指定してください

覚えておいてください：実質的な新しい情報がある場合にのみ更新してください。マジックドキュメントヘッダー（# MAGIC DOC: {{docTitle}}）は変更しないでください。
